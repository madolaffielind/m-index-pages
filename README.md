# m-index-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面可作为静态资源访问，适用于信息展示、工具页面、存档记录等场景。

## 项目简介

`m-index-pages` 是一个轻量级的页面归档仓库，主要功能是存放和管理若干独立的 HTML 文件。每个页面均为自包含的静态文档，不依赖外部数据库或后端服务。仓库结构简单，便于维护与扩展。

## 目录说明

```
m-index-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/`：仓库的核心目录，每个 HTML 文件代表一个独立页面。
- `assets/`：可选目录，用于存放页面共享的样式、脚本或媒体资源。若页面资源内嵌，此目录可为空。

## 页面归档说明

- 所有页面均为静态 HTML，可直接通过浏览器打开，或部署至任何静态托管服务。
- 页面文件名应简洁明了，建议使用英文小写字母、数字及连字符命名。
- 每个页面尽量保持独立，避免跨页面依赖（公共资源除外）。
- 如需添加新页面，请在 `pages/` 目录下创建对应的 HTML 文件，并确保其内容完整无误。

## 维护说明

- 本仓库欢迎贡献者提交新的 HTML 页面或改进现有页面。
- 提交前请检查页面是否能在主流浏览器中正常渲染。
- 请勿在页面中包含恶意代码、跟踪脚本或违反法律法规的内容。
- 建议为每个页面添加简洁的注释说明其用途和作者信息。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 目录下找到需要的 HTML 文件，双击即可在浏览器中打开。
3. 如需部署，可将整个仓库上传至静态托管平台（如 GitHub Pages、Netlify 等），并设置 `pages/` 为根目录。
